    持久化

    执行BGSAVE命令或者BGREWRITEAOF命令的过程中，Redis需要创建当前 服务器进程的子进程，而大多数操作系统都采用写时复制（copy-on-write）来 优化子进程的使用效率，所以在子进程存在期间，服务器会提高负载因子的阈值，从 而避免在子进程存在期间进行哈希表扩展操作，避免不必要的内存写入操作，最大限度地 节约内存。

    RDB

    • SAVE: 前台执行，生成dump.rdb文件，阻塞网络服务请求
    • BGSAVE: 后台执行，生成dump.rdb文件
    • LASTSAV: 查看BGSAVE是否成功
    save 100 1 //100秒内，至少更新了1条数据，Redis自动触发BGSAVE命令
save 200 100 //200秒内，至少更新了100条数据，Redis自动触发BGSAVE命令
save 60 1000 //60秒内，至少更新了1000条数据，Redis自动触发BGSAVE命令
